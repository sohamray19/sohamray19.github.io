τ²-bench · task construction

The compositional task generator

Tasks aren’t written — they’re initialized. Start from a healthy world and break it, composing independent root causes on either side:

The recipe

1  world = healthy_telecom()

2  world.apply(RC₁, RC₂)  ← same tools the user has

3  symptom emerges: “No Service”

4  gold_fix = the undo plan — recorded at build time

RC₂ is cross-world: “abroad” lives on the phone, “roaming off” in the CRM — only together do they break service. We injected the causes, so the fix provably exists.

τ-voice

Grounded tasks, full-duplex, over a realistic phone line

Voice is becoming a primary interface for agentic systems — and it is where text-benchmark performance goes to die.

Ray, Dhandhania, Barres & Narasimhan — arXiv 2603.13686 · ICML 2026

What we see in production

The call you design for vs. the call you get

How we model it

How do you build a representative user simulator?

You’d want the user to:

…act like a real callerThe turn-taking policy — barge-ins, backchannels, yields, hang-ups.
…sound like a real callerThe audio pipeline — persona TTS, accents, noise, telephony.

But no voice model today is strong enough to play this caller at human-level latency — you have to choose: a high-quality, controllable user sim, or a low-latency one.

How we model it

Wait — what if you could pause time?

How we model it · 1 of 3

Introducing ‘ticks’

The full-duplex orchestratorRuns the whole call on this tick grid — generating, slicing, exchanging, and timing tool calls.
Ticks are how you pause timeA tick is simulated time, not wall-clock time — the user sim has no latency budget, can be the strongest LLM available, and every run is reproducible.

The tick log — audio, transcript slice, tool calls, per tick — is the single source of truth every downstream metric is computed from.

Zoom in · turn-taking

Interrupts & backchannels, tick by tick

User-simulator defaults: respond after 1 s of agent silence · yield after 1 s when interrupted · hold 5 s when interrupting. All knobs — and deterministic given a seed.

Beyond task success

Interactivity metrics

Task reward says nothing about how it felt to talk to the agent. From the tick logs we score four dimensions of conversational competence:

DimensionWhat it captures
Responsiveness Does the agent answer each user turn — and stop within 2 s when barged-in?
Latency Time from end-of-user-speech to agent speech; time to go quiet after an interruption.
Interruption How often the agent speaks before the user finishes — can exceed 100% per turn.
Selectivity Correctly ignoring “mm-hmm”, coughs, and speech aimed at someone else in the room.

Next · work in progress

Multilingual: τ-voice for the other 80%

Q1Can we build a representative multilingual user simulator?
Q2Can we measure how agents fail — reliably, and beyond accuracy?

In multilingual, user experience matters: other languages carry nuances — grammatical gender, honorifics, register — that aren’t accounted for today.

What we’re seeing

Language is not translation

Pragmatics, not words Meet Adriana (Cluj, Romania): formal register, dumneavoastră throughout, refuses indirectly — and when the agent drifts into English she gently pushes back: “putem continua în română?” An agent that translates but doesn’t operate in the language fails her call.
Code-switching is a spectrum Hinglish callers mix 0–30% English mid-sentence. We built a density ladder to find where each model breaks — comprehension degrades long before generation does.
Honorifics are load-bearing Korean speech levels, Japanese keigo, T–V distinctions: get them wrong and the call feels broken even when the task succeeds. Task reward alone can’t see this — nativeness scoring can.
Cross-lingual knowledge The knowledge base stays English; the caller doesn’t. Reading English policy and answering natively is its own capability — we ablate it directly (KB-on/KB-off).
Within-family probes Egyptian ↔ Levantine Arabic, Hindi ↔ Bengali, Mandarin ↔ Cantonese: is a provider’s gap family-wide or language-specific? Early answer: it varies by provider — coverage maps are uneven.
Diglossia is the default Arabic callers don’t speak the written language: the KB is Modern Standard Arabic, the caller is Egyptian or Levantine. Reading fuṣḥā and speaking dialect — bridged mid-call — is the job.
Stepping back

Thoughts on benchmarking

What to benchmark

Distill production problems. LLMs will draft the tasks, worlds, even judges — the scarce input is taste: what’s worth measuring. Every τ axis is a production failure mode, distilled. And a benchmark is a steering wheel: we aim the labs at problems we know are tractable.

“In this new era, evaluation becomes more important than training.”
Shunyu Yao — τ-bench’s first author — The Second Half, 2025: the second half of AI shifts from solving problems to defining them.
“Effort shifts from building systems to evaluating systems.”
Arvind Narayanan — What will be left for us to work on?, ICML 2026 keynote — days ago.
How to benchmark

Reliability is half of quality. τ²’s accidental lesson: don’t prompt for a reliable simulator, provide a world. Grounding the user in real state cut critical simulator errors ~13% → 6% — realism made the measurement cleaner.

Verifiability is the other half. The core reward is a database diff — no judge can argue with it. Rubric rewards (delivery, intonation, nativeness) now see what state diffs can’t — but every rubric point is a point you can’t verify. How do you walk this tradeoff?

Yao, “The Second Half” (ysymyth.github.io, Apr 2025) · Narayanan, ICML 2026 keynote (Jul 13, 2026 — AI Snake Oil / normaltech.ai).

Appendix · related work

How the field evaluates conversational agents

Two parallel threads: task-grounded evaluation and conversational-dynamics evaluation. The Full-Duplex-Bench line traces the second:

Full-Duplex-Bench · v1–1.5Turn-taking probes. Pause handling, backchanneling, smooth handoffs, user interruptions — pre-recorded audio, one round, offline scoring. v1.5 adds overlapping speech.
Full-Duplex-Bench · v2Goes real-time. A WebRTC orchestrator and an automated AI examiner drive live multi-turn exchanges — dynamics measured in the wild, but still no task to complete.
Full-Duplex-Bench · v3Adds tool use. Full-duplex agents calling tools under real-world disfluency — converging on our thesis from the dynamics side.
Talking Turns & co.Judged dynamics. Turn-taking judged against human conversational norms; VoiceBench / VocalBench probe speech understanding — no state, no tools.

The two threads don’t meet: dynamics benchmarks have no verifiable task, task benchmarks have no live audio. τ-voice is built to sit at that intersection.

Appendix · the user simulator

Turn-taking is a controlled policy

A separate LLM policy runs on the user side during agent speech, deciding what a real caller would do with their voice right now:

Barge-inThe caller cuts in mid-sentence when the agent rambles or gets it wrong — the agent’s audio is truncated at the interruption point.
YieldingWhen both sides speak at once, configurable overlap thresholds decide who backs off — on either side of the line.
Backchannels“mm-hmm” · “okay” · “right” — overlap without taking the turn. Density is a per-persona knob.
Follow-upsIf the agent pauses too long, the caller speaks up: “hello? you still there?” — dead air gets probed, not tolerated.
Hang-upsKept on hold past the persona’s patience, the caller drops the call — and the task scores as a failure.

The turn-taking policy is evaluated every 2 s during agent speech; interrupt tendency, patience, and backchannel density are per-persona parameters.
